Understanding Multilevel Feedback Queues

You correctly identified that it uses multiple queues, but didn't mention how processes move between them based on CPU burst behavior.

lightbulb Model Answer

A multilevel feedback queue uses several queues, each with a different priority. New processes enter the top (highest-priority) queue, and the scheduler always runs jobs from the highest non-empty queue. If a process uses its full time slice it is demoted to a lower-priority queue; if it yields early (I/O-bound) it stays high. This lets the scheduler learn a job's behavior over time, favoring short, interactive tasks without knowing their run-time in advance.
